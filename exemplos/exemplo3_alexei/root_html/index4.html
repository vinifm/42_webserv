WEBSERV

Sobre o projeto

O projeto "webserv" é um projeto em C++ que consiste em implementar um servidor web que possa responder a solicitações HTTP de clientes e fornecer conteúdo estático e dinâmico.

O projeto começa com a criação de uma estrutura básica do servidor, implementando as funções para lidar com as conexões TCP e as solicitações HTTP básicas. Em seguida, deverá ser capaz de lidar com solicitações de arquivos estáticos, como HTML, CSS, etc.

A parte mais desafiadora do projeto é a implementação de solicitações de arquivos dinâmicos, como scripts CGI (Common Gateway Interface).

O grande objetivo é adquir uma compreensão profunda de como os servidores web funcionam e ganhar habilidades em C++ e em programação de redes.









Execução

Devido à complexidade do projeto, o mesmo foi dividido basicamente em 4 etapas:

  • Criação das páginas
  • Determinação dos “Requests” e “Parsing”
  • Execução dos “Responses”
  • Testes

    • O grande aprendizado deste projeto foi a forma de desenvolvimento, as coisas só começaram a fluir quando marcamos reuniões diárias para acompanhamento da evolução de todos os integrantes.

    SIEGE é um programa de teste de estresse de rede de código aberto que permite aos usuários simular tráfego de usuários simultâneos em um servidor web ou aplicação para avaliar sua capacidade de lidar com a carga.

    O SIEGE pode ser usado para testar servidores web, aplicativos web, bancos de dados e outros serviços de rede. Ele é capaz de simular centenas ou milhares de usuários simultâneos e fornecer estatísticas detalhadas sobre o desempenho do servidor, incluindo tempos de resposta, taxas de erro, tempo de conexão, etc.

    O SIEGE é um software livre e pode ser baixado gratuitamente por qualquer pessoa que queira testar o desempenho de um servidor ou aplicativo web. Ele está disponível para sistemas operacionais Unix-like, incluindo Linux, macOS e FreeBSD.







    Um HTTP Server (Servidor HTTP) é um software (ou um componente) que recebe e responde a solicitações HTTP (Protocolo de Transferência de Hipertexto) de clientes. Ele funciona como uma ponte entre o cliente e o servidor, e é responsável por processar as solicitações do cliente, buscar os recursos solicitados e enviar a resposta de volta ao cliente.

    O funcionamento básico de um HTTP Server é simples: ele aguarda a chegada de solicitações HTTP de um cliente, processa a solicitação e, em seguida, envia a resposta de volta ao cliente. A resposta pode incluir dados, arquivos, imagens, páginas da web, entre outros.

    Existem muitos servidores HTTP disponíveis no mercado, como o Apache, Nginx, IIS, Lighttpd, entre outros. Esses servidores têm diferentes recursos e capacidades e podem ser configurados de várias maneiras para atender às necessidades específicas de um aplicativo ou site da web.







    I/O Multiplexing é uma técnica de programação que permite que um processo monitore vários descritores de arquivos (como sockets e pipes) ao mesmo tempo em busca de eventos de entrada ou saída. Isso significa que o processo pode lidar com vários fluxos de entrada e saída sem bloquear enquanto espera que cada um seja concluído.

    A técnica de I/O Multiplexing é comumente usada em aplicativos de rede que precisam lidar com muitas conexões simultâneas, como servidores web, de jogos, de chat e outros aplicativos que precisam responder a várias solicitações ao mesmo tempo.

    Em vez de usar uma abordagem de programação sequencial, em que um processo espera até que uma solicitação seja concluída antes de passar para a próxima, o I/O Multiplexing usa um modelo de programação baseado em eventos. Quando um evento de entrada ou saída é detectado em um descritor de arquivo, o processo pode ser notificado e realizar a operação necessária de maneira assíncrona, sem precisar esperar que a operação anterior seja concluída.

    Algumas das interfaces de programação mais comuns usadas para implementar a técnica de I/O Multiplexing incluem o select(), poll() e epoll() no sistema operacional Linux, o kqueue() no macOS e o IOCP (Input/Output Completion Ports) no Windows.







    É uma função de sistema em sistemas operacionais baseados em Unix que permite que um programa monitore múltiplos descritores de arquivo (como sockets e arquivos) para eventos de entrada ou saída. É uma das interfaces de programação mais comuns usadas para implementar a técnica de I/O Multiplexing.

    A função poll() aceita um array de estruturas de dados chamadas "estruturas de evento" (struct pollfd) que contêm informações sobre os descritores de arquivos a serem monitorados. O programa pode especificar qual tipo de eventos (leitura, escrita, exceção) deseja monitorar para cada descritor de arquivo.

    Quando um evento de entrada ou saída é detectado em um dos descritores de arquivo monitorados, a função poll() retorna e fornece informações sobre quais descritores de arquivo tiveram eventos de entrada ou saída. Isso permite que o programa tome as ações necessárias em relação a cada evento detectado.

    Ao contrário do select(), a função poll() não tem limitação no número máximo de descritores de arquivo que pode monitorar. Isso a torna uma escolha popular para aplicativos que precisam lidar com muitas conexões simultâneas.







    Cookies e sessões são mecanismos usados em aplicações web para armazenar informações do usuário e rastrear atividades do usuário em diferentes sessões, são amplamente utilizados para fornecer uma experiência personalizada e melhorar a usabilidade do site. No entanto, é importante que as informações armazenadas em cookies e sessões sejam protegidas para evitar violações de privacidade e segurança.

    Um cookie é um pequeno arquivo de texto que um servidor envia para o navegador do usuário. O navegador armazena o cookie no computador do usuário e envia-o de volta para o servidor sempre que o usuário solicita uma página. Os cookies geralmente são usados para armazenar informações de login, preferências do usuário e outras informações personalizadas.

    As sessões, por outro lado, são objetos que permitem que o servidor armazene informações específicas do usuário durante uma sessão de navegação. A sessão é geralmente iniciada quando o usuário faz login no site e termina quando o usuário fecha o navegador ou faz logout. As sessões são frequentemente usadas para manter o estado do usuário e permitir que o servidor rastreie atividades específicas do usuário, como o carrinho de compras em um site de e-commerce.

    As principais diferenças entre cookies e sessões são:

  • Cookies são armazenados no computador do usuário, enquanto as sessões são armazenadas no servidor.
  • Cookies são geralmente usados para armazenar informações persistentes, enquanto as sessões são usadas para armazenar informações temporárias.
  • Cookies são limitados em tamanho (geralmente cerca de 4 KB), enquanto as sessões podem armazenar informações maiores.






    • CGI (sigla para Common Gateway Interface) é um protocolo que permite que programas executáveis em um servidor web gerem conteúdo dinâmico em resposta a uma solicitação de um cliente (geralmente um navegador). O programa CGI é executado no servidor para gerar uma resposta dinâmica, que é então enviada ao cliente pelo servidor web.

    O protocolo CGI define um conjunto de padrões para a comunicação entre o servidor web e o programa CGI. Quando o cliente solicita um arquivo CGI (geralmente um arquivo com extensão .cgi ou .pl), o servidor web executa o programa CGI e passa informações relevantes (como informações de formulário enviadas pelo cliente) por meio de variáveis de ambiente e/ou parâmetros de linha de comando. O programa CGI pode então processar essas informações e gerar uma resposta personalizada para o cliente, geralmente no formato de uma página HTML.

    Os programas CGI podem ser escritos em várias linguagens de programação, como Perl, Python, C, C++, Java e outras. No entanto, o uso de CGI tem algumas desvantagens, como a sobrecarga do servidor e a falta de escalabilidade em grandes volumes de tráfego. Por esse motivo, outras tecnologias, como PHP, ASP.NET e Java Servlets/JSPs, foram desenvolvidas para melhorar o desempenho e escalabilidade na geração de conteúdo dinâmico em servidores web.
